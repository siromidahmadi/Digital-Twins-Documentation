Modular Architecture

Digital Twins Project is organized as small, focused modules that communicate via well-defined interfaces and standardized data formats (GeoJSON, time-series JSON).

Ingest Module

Function: Pulls feeds (USGS, FIRMS, OpenWeather), normalizes to internal GeoJSON schema, queues updates.

Data Types: GeoJSON, CSV

Problem solved: Unifies multiple source formats and makes them queryable.

API Layer

Function: Offers endpoints for filtered data, vector tiles, and precomputed statistics.

Data Types: GeoJSON, JSON

Problem solved: Provides performant access for visualization and analytics.

Visualization

Function: Renders globe, vector routes, annotations, and interactive controls.

Data Types: Vector tiles, raster tiles

AI Assistant

Function: Interpret user queries, map to geospatial actions, and provide human-friendly explanations.

Data Types: Text, JSON

Function of Each Module (Concise)

  • Ingest: Fetch & normalize data.
  • Storage: Store time series, tile caches, and metadata.
  • API: Serve data & analytics endpoints.
  • UI/Vis: Present maps, controls, and user photo.
  • Assistant: Handle NL queries & guidance.
